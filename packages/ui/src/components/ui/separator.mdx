## Overview

The Separator component visually divides content into sections without using borders or extra spacing.
It supports both horizontal and vertical orientations and follows accessibility guidelines.

### Key Features

- **Horizontal & Vertical**: Supports both orientations
- **Accessible**: Proper ARIA roles for semantic meaning
- **Decorative Mode**: Can be marked as decorative for screen readers
- **Customizable**: Easy to style with Tailwind classes
- **Semantic HTML**: Uses appropriate ARIA separator role

## Accessibility

The Separator component follows WCAG 2.1 Level AA guidelines:

- ✅ **ARIA Role**: Proper separator role for semantic meaning
- ✅ **Decorative Option**: Can be marked as decorative
- ✅ **Orientation**: Properly indicates horizontal or vertical orientation
- ✅ **Screen Reader Support**: Announced appropriately or ignored if decorative

### Best Practices

1. **Use sparingly**: Too many separators can be visually overwhelming
2. **Consider spacing**: Sometimes margin/padding is better than a separator
3. **Mark as decorative**: If purely visual, set decorative={true}
4. **Match orientation**: Ensure orientation matches layout (horizontal/vertical)
5. **Consistent styling**: Keep separator style consistent across your app

```tsx
// For semantic separation
<Separator decorative={false} />

// For purely visual decoration (default)
<Separator decorative={true} />
```

## When to Use Separators

Use separators for:
- **Content sections**: Dividing distinct content areas
- **Menu items**: Grouping related menu options
- **List items**: Separating items in vertical lists
- **Inline elements**: Dividing breadcrumbs, metadata, etc.

Don't use separators when:
- **Spacing alone works**: Sometimes margin is enough
- **Overuse**: Too many separators reduce their effectiveness
- **Card headers**: Often better handled with CardHeader component
